# From Zero to Production: Deploying Go APIs on AWS with Docker

Go's compilation speed, small binary sizes, and excellent concurrency primitives make it perfect for production APIs. In this post, I'll walk through the complete journey from writing a Go API to running it in production on AWS — the same approach I used for **Jeanpay Africa's** currency exchange backend.

## Why Go for APIs?

- **Single binary deployment** — no runtime dependencies
- **Built-in concurrency** — goroutines handle thousands of concurrent requests efficiently
- **Fast compilation** — sub-second builds even for large projects
- **Low memory footprint** — perfect for containerized environments

## Docker Multi-Stage Build

The magic of Go + Docker is the multi-stage build. Your final image can be under 20MB:

```dockerfile
# Build stage
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o /api ./cmd/api

# Production stage
FROM alpine:3.19
RUN apk --no-cache add ca-certificates
COPY --from=builder /api /api
EXPOSE 8080
CMD ["/api"]
```

## AWS Deployment Strategy

For Jeanpay Africa, we use EC2 instances behind an Application Load Balancer:

1. **ECR** for Docker image registry
2. **EC2 Auto Scaling Group** for the API servers
3. **RDS PostgreSQL** for the database
4. **ElastiCache Redis** for caching and sessions
5. **CloudWatch** for monitoring and alerts

## CI/CD with GitHub Actions

```yaml
name: Deploy API
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build and push to ECR
        run: |
          aws ecr get-login-password | docker login --username AWS --password-stdin $ECR_URL
          docker build -t $ECR_URL/api:${{ github.sha }} .
          docker push $ECR_URL/api:${{ github.sha }}
      - name: Deploy to EC2
        run: |
          ssh ec2-user@${{ secrets.EC2_HOST }} "
            docker pull $ECR_URL/api:${{ github.sha }}
            docker stop api || true
            docker run -d --name api -p 8080:8080 $ECR_URL/api:${{ github.sha }}
          "
```

## Performance Results

After migrating Jeanpay's exchange rate service from Node.js to Go:

- **Response time**: 45ms → 8ms (p99)
- **Memory usage**: 512MB → 64MB
- **Docker image**: 350MB → 18MB
- **Throughput**: 2x more requests per instance

## Key Takeaways

1. **Multi-stage Docker builds** are essential for Go — keep production images minimal
2. **Health checks** should be built into the API from day one
3. **Graceful shutdown** handling prevents dropped connections during deployments
4. **Structured logging** (JSON) makes CloudWatch analysis straightforward
